# Static Website Editing Guide

This website is a static Jekyll site based on the al-folio theme. Most day-to-day edits are made in Markdown, YAML, BibTeX, image files, and a small SCSS stylesheet. Do not edit `_site/` directly, because it is generated by Jekyll and will be overwritten.

## Editing the Homepage / About Page

File:

```text
_pages/about.md
```

Common edits:

- `subtitle`: position or short description below the page title.
- `profile.image`: profile image filename. The image should be stored in `assets/img/`.
- `profile.more_info`: address, email, phone, and profile links.
- Main body paragraphs: homepage introduction.
- `Research themes`: research topic list.
- `Teaching and community`: teaching and service description.

Example:

```yaml
profile:
  image: mariana-belgiu.jpg
```

This points to:

```text
assets/img/mariana-belgiu.jpg
```

## Editing the Research Page

Main page:

```text
_pages/projects.md
```

Project card files:

```text
_projects/data-centric-eo.md
_projects/hidden-hunger.md
_projects/urban-vulnerability.md
_projects/eo-education-inclusion.md
```

To add a new research project, create a new `.md` file inside `_projects/`. Typical front matter:

```yaml
---
layout: page
title: Project title
description: Short description
img: assets/img/group-placeholder.png
importance: 1
category: research
---
```

Lower `importance` values appear earlier.

## Editing the Publications Page

Page file:

```text
_pages/publications.md
```

This file contains:

- The Google Scholar-style citation summary.
- The `VIEW ALL` citation chart modal.
- The public access block.
- The bibliography search and publication list.

Publication data file:

```text
_bibliography/papers.bib
```

Add a new paper by inserting a BibTeX entry into `papers.bib`. Example:

```bibtex
@article{example2026paper,
  bibtex_show = {true},
  abbr        = {Journal},
  title       = {Paper title},
  author      = {Belgiu, Mariana and Other, Author},
  journal     = {Journal Name},
  year        = {2026},
  selected    = {true}
}
```

`selected = {true}` also shows the paper in the selected papers section on the homepage.

Venue abbreviations and colors can be edited in:

```text
_data/venues.yml
```

```text
_pages/profiles.md
```

```text
_pages/people_director.md
```

```text

```

Common edits:

- Edit member names.
- Edit study periods or graduation periods.
- Edit research interests.
- Add or update email addresses.
- Replace portrait images.

Put portrait images in:

```text
assets/img/
```

Then reference them like this:

```liquid
{{ '/assets/img/your-image.jpg' | relative_url }}
```

## Editing News

News page:

```text
_pages/news.md
```

News items:

```text
_news/announcement_1.md
_news/announcement_2.md
_news/announcement_3.md
```

To add a news item, create a new `.md` file inside `_news/`. Example:

```yaml
---
layout: post
title: News title
date: 2026-06-29 09:00:00+0200
inline: false
related_posts: false
---

News content here.
```

Use `inline: true` for short announcement-style items.

## Editing the Navigation Bar

The navigation bar is controlled by front matter fields in each page:

```yaml
nav: true
nav_order: 3
```

Meaning:

- `nav: true`: show this page in the navigation bar.
- `nav_order`: navigation order. Lower numbers appear earlier.
- `title`: text shown in the navigation bar.

Main pages:

```text
_pages/about.md          /
_pages/projects.md       /research/
_pages/publications.md   /publications/
_pages/profiles.md       /teams/
_pages/news.md           /news/
_pages/teaching.md       /teaching/
```

## Editing Styles

Main custom style file:

```text
_sass/_site.scss
```

Stylesheet entry point:

```text
assets/css/main.scss
```

In most cases, edit only `_sass/_site.scss`. It currently contains custom styling for:

- Homepage spacing and section separators.
- Research project highlights.
- Teams member cards.
- Publications citation summary and modal chart.
- Teaching page layout.

Do not edit files inside the theme gem or `vendor/`.

## Editing Global Settings

File:

```text
_config.yml
```

Common fields:

```yaml
title: Mariana Belgiu
description: ...
keywords: ...
max_width: 1080px
url: http://localhost:4000
baseurl:
```

For GitHub Pages, update `url` and `baseurl` depending on the repository type.

User or organization site:

```yaml
url: https://your-username.github.io
baseurl:
```

Project site:

```yaml
url: https://your-username.github.io
baseurl: /repository-name
```

## Files You Should Not Edit Manually

Do not manually edit:

```text
_site/
vendor/
.jekyll-cache/
```

Why:

- `_site/` is generated output and will be overwritten.
- `vendor/` contains Ruby dependencies.
- `.jekyll-cache/` contains Jekyll cache files.
